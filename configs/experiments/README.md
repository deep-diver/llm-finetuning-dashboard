# Active Experiments Configuration Directory

This directory stores run-specific configs generated by the `training-configuration` agent.

Files are dynamically created by merging profiles from `configs/base/` with overrides proposed during active research iterations.

## Naming convention
`configs/experiments/<run_id>.yaml`

## Structure
Each experiment configuration consolidates:
- `run_id`
- Model settings & overrides
- Dataset overrides & path points
- Hyperparameters for local and full training runs
- Target TPU VM configurations
- Target Evaluation settings
